1. INTRODUCTION TO MYSQL & RDBMS
-----------------------------------------------------------------------
* Definition: MySQL is an easy-to-use, open-source Relational Database 
  Management System (RDBMS). It helps you store, organize, and manage 
  data using Structured Query Language (SQL).
* Key Features: It is completely free, runs on multiple systems (Windows, 
  Mac, Linux), is very fast, and works great for both small and large apps.
* Database: A simple folder or container that holds your tables.
* Table: A grid made of vertical columns (fields) and horizontal rows 
  (records) that hold your actual information.

* Relational vs. Non-Relational:
  - Relational (RDBMS): Stores data in clean tables with rows and columns. 
    It uses a fixed structure and links tables together (like MySQL, Oracle).
  - Non-Relational (NoSQL): Does not use tables. It stores data in flexible 
    formats like files, maps, or graphs (like MongoDB, Redis).


2. BASIC COMMANDS & CRUD OPERATIONS
-----------------------------------------------------------------------
show databases;          -- Shows all databases on your server
create database animal;  -- Creates a brand new database named 'animal'
use batch1182;           -- Switches your workspace to 'batch1182'
show tables;             -- Shows all tables inside the active database

  select curtime();  -- Shows the current time
  select database(); -- Shows the name of the database you are using right now
  select user();     -- Shows your current login username
  select curdate();  -- Shows today's date
  select now();      -- Shows both today's date and the current time

* Basic CRUD Operations (Create, Read, Update, Delete):
  desc student;         -- Shows the structure/columns of the student table
  select * from student;-- Grabs and displays all records from the student table

  -- Create a table:
  create table employee(id int, name varchar(30), address varchar(50));
  
  -- Insert a single row:
  insert into employee values(1, 'shubham', 'pune');
  
  -- Insert multiple rows at once:
  insert into employee(id, name, address) values
  (2, 'abhishek', 'pune'),
  (3, 'nisha', 'nadurbar'),
  (4, 'lochana', 'jalgaon');

  -- Update a record:
  update employee set address = 'pune' where id = 2;
  
  -- Delete a record:
  delete from employee where id = 1;
  
  exit; -- Closes the MySQL command line connection


3. CONSTRAINTS & DATA TYPES
-----------------------------------------------------------------------
* PRIMARY KEY : Uniquely identifies each row. It cannot be blank (NULL) and 
  cannot have duplicates. You only get one primary key per table.
* NOT NULL    : Forces a column to always have a value. It cannot be left blank.
* UNIQUE      : Makes sure all values in a column are completely different.
* AUTO_INCREMENT: Automatically numbers your rows (1, 2, 3...) as you add them.

* Dealing with NULL Values:
  - A NULL value means the field was left completely blank. It is not the 
    same as a zero or an empty space!
  - Query: select * from employee where email IS NULL;
  - Query: select * from employee where email IS NOT NULL;

* Common Date Data Types:
  - DATE      : Format YYYY-MM-DD (e.g., 2024-12-23)
  - DATETIME  : Format YYYY-MM-DD HH:MI:SS (e.g., 2024-09-26 12:25:14)
  - TIMESTAMP : Format YYYY-MM-DD HH:MI:SS (Changes based on time zones)
  - YEAR      : Format YYYY or YY (e.g., 2024 or 24)


4. MODIFYING TABLES, SORTING, & FILTERS
-----------------------------------------------------------------------
* ALTER TABLE: Used when you need to change the structure of an existing table.
  - Add a new column: alter table employee add country varchar(50);
  - Modify a column type: alter table employee modify mob_numb int;
  - Delete a column: alter table employee drop column email;
  - Rename a column: alter table employee rename column email to gmail;
  - Rename a whole table: alter table employee rename to employeeRecord;

* ORDER BY: Sorts your data. It sorts from smallest to largest (ASC) by default.
  - select * from employeerecord order by salary desc; -- Sorts largest to smallest

* LIMIT & OFFSET: Limits the number of rows returned so your database stays fast.
  - select * from Employee limit 3 offset 3; -- Skips the first 3 rows, shows rows 4-6

* ALIAS (AS): Gives a column or table a temporary, easier-to-read nickname.
  - select email as employee_email from employeerecord;

* SQL COMMENTS:
  - Single Line Comment: Starts with '--'. MySQL will ignore this line.
  - Multi-line Comment : Starts with '/*' and ends with '*/'.


5. OPERATORS & PATTERN MATCHING
-----------------------------------------------------------------------
* AND / OR / NOT: Used to mix multiple conditions together.
  - select * from employee where emp_id = 101 and name = 'ritik';
  - select * from employee where not salary = 300000;

* IN Operator: A quick shortcut instead of writing multiple 'OR' statements.
  - select * from empdata where State in ('maharashtra', 'up', 'mp');

* BETWEEN Operator: Selects values within a specific range (includes the start 
  and end values).
  - select * from employeerecord where age between 10 and 20;

* LIKE Operator: Used to search for specific text patterns using two wildcards:
  - '%' means zero, one, or multiple characters.
  - '_' means exactly one single character.
  - Pattern Examples:
    's%'   -> Finds any values that start with "s"
    '%h'   -> Finds any values that end with "h"
    '%sh%' -> Finds any values that have "sh" anywhere inside
    '_a%'  -> Finds any values that have "a" in the second position


6. ADVANCED AGGREGATIONS & SUBQUERIES
-----------------------------------------------------------------------
* Aggregate Functions: Take a whole list of numbers and calculate a single answer.
  - MIN() (Smallest), MAX() (Largest), COUNT() (Total rows), AVG() (Average), 
    SUM() (Total sum). They all skip blank (NULL) values.

* GROUP BY & HAVING:
  - GROUP BY gathers rows that have identical data into simple groups.
  - HAVING works just like a WHERE clause, but it is strictly used to filter 
    groups after they are made.
  - Query: select CourseName, count(CourseName) as aaa from tkastd group by CourseName having avg(totalFees) > 45000;

* Subqueries: A query written inside another query ("Query inside a query").
  - Example: Finding the employee with the highest salary:
    select name, salary from employees where salary = (select max(salary) from employees);


7. FOREIGN KEYS & JOINS
-----------------------------------------------------------------------
* FOREIGN KEY: A column in one table that points directly to a Primary Key 
  in another table. This links them together securely.



* Simple Join Descriptions:
  1. INNER JOIN: Only shows rows where there is a clean match in both tables.
  2. LEFT JOIN : Shows every single row from the left table, plus any matching 
     rows from the right table.
  3. RIGHT JOIN: Shows every single row from the right table, plus any matching 
     rows from the left table.
  4. FULL OUTER JOIN: Shows all records from both tables and leaves blanks 
     where there is no match.
  5. CROSS JOIN: Pairs every row from the first table with every single row 
     from the second table.
  6. SELF JOIN : Joins a table with itself to compare rows in that same table 
     (like matching a worker to their boss).

-- Table Setup Script:
create table departments (
    dept_id int primary key,
    dept_name varchar(50)
);

create table employees (
    emp_id int primary key,
    name varchar(50),
    dept_id int,
    manager_id int,
    foreign key (dept_id) references departments(dept_id)
);

-- Sample Data:
insert into departments (dept_id, dept_name) values 
(1, 'HR'), (2, 'Engineering'), (3, 'Finance'), (4, 'Marketing');

insert into employees (emp_id, name, dept_id, manager_id) values 
(101, 'Alice', 1, null),
(102, 'Bob', 2, 101),
(103, 'Charlie', 2, 102),
(104, 'David', 3, null),
(105, 'Eve', null, 101);


1. INNER JOIN
-----------------------------------------------------------------------
SELECT employees.name, departments.dept_name
FROM employees
INNER JOIN departments
ON employees.dept_id = departments.dept_id;


2. LEFT JOIN
-----------------------------------------------------------------------
SELECT employees.name, departments.dept_name
FROM employees
LEFT JOIN departments
ON employees.dept_id = departments.dept_id;


3. RIGHT JOIN
-----------------------------------------------------------------------
SELECT employees.name, departments.dept_name
FROM employees
RIGHT JOIN departments
ON employees.dept_id = departments.dept_id;


4. FULL OUTER JOIN
-----------------------------------------------------------------------
SELECT employees.name, departments.dept_name
FROM employees
FULL OUTER JOIN departments
ON employees.dept_id = departments.dept_id;


5. CROSS JOIN
-----------------------------------------------------------------------
SELECT employees.name, departments.dept_name
FROM employees
CROSS JOIN departments;


6. SELF JOIN
-----------------------------------------------------------------------
SELECT e1.name AS Employee, e2.name AS Manager
FROM employees e1
INNER JOIN employees e2
ON e1.manager_id = e2.emp_id;

CONSTRAINTS CHEATSHEET:
* NOT NULL       -> No empty boxes allowed
* UNIQUE         -> No duplicate entries allowed
* PRIMARY KEY    -> Unique + Not Null (The perfect ID row marker)
* FOREIGN KEY    -> Connects this table to an external table
* CHECK          -> Checks if data fits a basic rule
* DEFAULT        -> Adds a backup value if you leave the field blank
* AUTO_INCREMENT -> Automatically handles numbering items for you

ORDER OF SQL CLAUSES:
1. SELECT     -> Which columns do you want to see?
2. FROM       -> Which table are you looking at?
3. JOIN       -> Are you pulling data from other connected tables?
4. WHERE      -> How do you want to filter individual rows?
5. GROUP BY   -> How do you want to group your rows?
6. HAVING     -> How do you want to filter your groups?
7. DISTINCT   -> Do you want to remove duplicate results?
8. ORDER BY   -> How do you want to sort the final list?
9. LIMIT      -> How many rows maximum do you want to return?
